# Yani_Montagenik - Визитка

Это минималистичная страница-визитка для Yani_Montagenik, адаптированная для GitHub Pages.
Содержит все актуальные ссылки и информацию профиля без использования сторонних конструкторов.

## Структура проекта

- `index.html` - основная страница с социальными ссылками
- `style.css` - стили для красивого отображения страницы
- `assets/` - директория с ресурсами (изображениями)
- `CNAME` - файл для настройки кастомного домена GitHub Pages
- `README.md` - этот файл документации

## Особенности дизайна

- Адаптивный дизайн, работает на мобильных устройствах
- Современные стили с градиентами и анимациями
- Все ссылки из оригинальной страницы сохранены
- Используется локальный аватар из assets

## Как использовать

1. Создайте репозиторий на GitHub и загрузите в него содержимое директории `yanimontagenik.ru`.
2. В настройках репозитория (Settings → Pages) выберите ветку, из которой будет публиковаться сайт (обычно `main` / `/root`).
3. Дождитесь, пока GitHub Pages соберёт сайт, и откройте выданный URL, чтобы убедиться, что визитка работает.
4. Для кастомного домена укажите его в настройках GitHub Pages и создайте DNS‑записи у регистратора:
   - `CNAME` для поддомена (например, `yanimontagenik.ru` → `ВАШЕ_ИМЯ_ПОЛЬЗОВАТЕЛЯ.github.io`);
   - при необходимости `A`‑записи на IP GitHub Pages (если используете корневой домен).
5. Убедитесь, что файл `CNAME` в репозитории содержит ваш домен (например, `yanimontagenik.ru`) — это закрепит домен за этим сайтом.
